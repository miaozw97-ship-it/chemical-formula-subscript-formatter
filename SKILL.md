# 全文下标审查与受控修订

先生成候选清单，再修改已确认项，最后复核。只改变字符的下标呈现，不改写字符、正文语义、数值、公式内容或编号。

## Agent 兼容性

- 本 skill 适用于任何能够读取 `SKILL.md`、访问用户文件并输出报告或修改副本的 AI Agent。
- 不依赖任何特定 Agent 平台、插件、文档编辑器或其他 skill。
- Agent 可使用自带工具或本目录脚本；无法可靠保留文件结构时只能审计，不得强行修改。
- 文中出现的命令、文件格式和流程要求描述的是能力与结果，不限定具体 Agent 平台。

## 必须输入

开始前确认：

1. 提供待处理文件和格式：DOCX、Markdown、LaTeX 或 TXT。
2. 说明是否处理数学公式、表格、图题/表题、脚注、参考文献表等区域；未说明时按保守默认值执行。
3. 提供项目中的专有材料式、变量约定或术语表（如有）。
4. 明确期望输出格式和路径。

文件缺失、格式不可编辑或无法可靠区分下标语义时，只生成可疑项清单并请求确认，不得猜测后批量修改。

## 格式与保守默认值

| 格式 | 下标实现 | 默认处理 |
|---|---|---|
| DOCX | Word 真实下标 `w:vertAlign=subscript` | 处理普通正文和用户允许的表格；跳过域代码、超链接、修订、批注锚点、文本框等复杂结构 |
| Markdown/TXT | 优先按用户要求；无要求时使用 Unicode 下标并报告兼容性限制 | 跳过代码块、行内代码、URL、文件路径 |
| LaTeX | 在数学/化学宏语境中使用 `_` 或既有宏风格 | 不用 Unicode 替换；不改命令、标签、引用键 |
| PDF | 仅审计 | 不直接修改 |

对 DOCX 不得把 Unicode 下标字符冒充 Word 真实下标。对 LaTeX 不得把普通文本规则直接应用到数学环境。

## 分类规则

### A. 高置信度化学/材料下标候选

- 元素后的化学计量数字：`Al2O3`、`Fe2O3`、`Fe3O4`、`O2`、`CO2`、`SO2`。
- 括号基团后的数字：`Ca(OH)2`、`Fe(NO3)3`。
- 掺杂和非整比材料式：`La0.8Sr0.2FeO3`、`CaMnTiFeOx`、`NOx`、`CeO2-x`、`O3-δ`。
- 处理结果必须保留原字符，仅改变应为下标的数字、`x`、`δ` 或缺陷项格式。
- `CuO` 等不含显式下标字符的正确化学式保持不变。

### B. 必须区分并默认不自动修改

| 类别 | 示例 | 处理 |
|---|---|---|
| 离子电荷/价态 | `Fe3+`、`Cu2+`、`O2−`、`Fe(III)` | 数字/符号通常应为上标或保持既有风格；列入人工复核，不改为下标 |
| 数学变量下标 | `x_i`、`a1`、`T2`、`k_app` | 仅在数学语境和用户规则明确时处理 |
| 单位与量值 | `900 ℃`、`2 h`、`10 wt%`、`50 mL`、`m2/g`、`cm-1` | 不按化学式规则修改；指数可能是上标 |
| 图表/章节编号 | `图2-1`、`表3.2`、`第3章`、`2.3.4` | 不修改 |
| 公式编号 | `式(2-3)`、`(4.1)` | 不修改 |
| 参考文献编号 | `[1]`、`[2-5]`、`\cite{ref2}` | 不修改 |
| 年份/页码/标识符 | `2024`、`pp. 12-15`、DOI、URL、文件名 | 不修改 |
| 反应系数 | `2Fe + O2 -> 2FeO` 中前置 `2` | 不修改系数；只审查化学式内部下标 |
| 晶相名称 | `α-Fe2O3`、`γ-Al2O3` | 相标识 `α/γ` 不作为下标；仅处理化学式内部数字 |

任何候选同时符合多个类别时，按低置信度处理并进入人工复核清单。

## 硬性禁止

- 禁止不经候选审计直接全文正则替换。
- 禁止改变正文字符、拼写、数值、标点、空格或语义来“修复”下标。
- 禁止把年份、温度、页码、图号、表号、章节号、公式号、参考文献号改成下标。
- 禁止把离子电荷、价态、单位指数或数学变量误改为化学式下标。
- 禁止重建 DOCX 整段文本而破坏样式、域代码、超链接、批注、修订、书签或公式。
- 禁止覆盖原文件；必须输出副本或补丁。
- 禁止把无法可靠处理的跨 run 或复杂结构候选静默忽略；必须报告。

## 三阶段执行流程

### 阶段 1：只读审计

1. 记录源文件路径和处理范围；进入阶段 2 前再创建备份/副本。
2. 扫描全文，按位置输出：
   - 高置信度可修改项；
   - 低置信度/歧义项；
   - 明确排除项；
   - 已正确格式化项；
   - 无法安全修改的复杂结构项。
3. 对每个候选显示原文、建议格式、位置、分类和置信度。
4. 抽查 `Al2O3`、`Fe2O3`、`CaMnTiFeOx`、`O2`、`CO2`、`SO2`、`NOx`、`CuO`、`Fe3O4` 是否分类正确。

阶段 1 不修改文件。没有候选清单，不得进入阶段 2。

### 阶段 2：受控修改

1. 仅修改用户确认的类别，或脚本中明确定义的高置信度候选。
2. DOCX 只修改能安全定位到普通文本 run 的字符；跨 run 或复杂 XML 结构候选默认跳过并报告。
3. Markdown/TXT 跳过代码、链接、路径；LaTeX 遵循现有宏与数学环境。
4. 输出新文件或补丁，不覆盖原文件。

### 阶段 3：复核

1. 重新扫描修改后的文件，确认目标候选已处理。
2. 比较修改前后可见文本，必须逐字符一致；只允许格式属性变化。Markdown/TXT 使用 Unicode 时，报告字符替换清单作为例外。
3. 检查章节、图表、公式、引用、年份、单位、离子价态、数学变量未被误改。
4. DOCX 抽查字体、字号、粗斜体、域代码、超链接、批注、修订、书签、公式和页眉页脚。
5. 发现误改时回退该项，不宣称完成。

## 脚本使用

`scripts/format_subscripts.py` 默认只审计：

```bash
python scripts/format_subscripts.py thesis.docx
python scripts/format_subscripts.py notes.md
```

审阅候选后，显式允许修改高置信度项：

```bash
python scripts/format_subscripts.py thesis.docx --apply --output thesis_subscript_fixed.docx
```

脚本只是保守辅助工具。它跳过的跨 run、复杂 DOCX 结构、数学变量、离子价态和歧义项必须人工复核。

## 与参考文献 Skill 配合

与 `seu-phd-thesis` 处理同一论文时：

1. 先完成引用编号修改，再运行下标审计。
2. 本 skill 默认排除正文引用标记和参考文献表，不改变引用编号。
3. 两个 skill 使用不同输出副本/补丁。
4. 最终同时复核引用映射、化学式下标和可见文本一致性。

## 最终输出

必须包含：

1. 修改摘要：处理范围、格式模式、输出文件。
2. 变更清单：位置、原文、建议/修改后格式、分类、置信度。
3. 风险提示：歧义语境、复杂 DOCX 结构、Unicode/LaTeX 兼容性。
4. 无法处理项：逐项给出位置与原因。
5. 建议人工复核项：离子、变量、单位指数、跨 run 和专有材料式。
6. 修改后的文件或补丁；未获修改条件时只输出审计报告。
7. 最终自检结果：
   - 典型化学式分类正确；
   - 非目标数字未被修改；
   - 离子/数学/单位/编号未误改；
   - 正文语义和可见文本未改变；
   - DOCX 结构未受破坏；
   - 所有跳过项均已报告。
